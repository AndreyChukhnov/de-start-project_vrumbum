Модель состоит из 6 таблиц. 5 таблиц — это справочники, а 1 таблица представляет собой таблицу событий (покупки авто).

Описание типов данных таблиц:

Таблица colors имеет следующие атрибуты:
	id_color smallserial PRIMARY KEY (тип данных smallserial обусловлен тем, что количество цветов для автомобилей невелико и вряд ли превысит максимальное значение в 32767).
	color varchar (для атрибута color предусмотрен тип varchar для хранения текстовых значений цветов).
Таблица cars содержит наименование марки автомобиля. Таблица имеет следующие атрибуты:
	id_car smallserial PRIMARY KEY (тип данных smallserial для автоматического формирования PK; размер таблицы не должен превышать 32767 строк).
	car varchar (для атрибута car предусмотрен тип varchar для хранения текстовых значений марок автомобилей).
Таблица models служит для описания моделей автомобилей и их технических характеристик, зависящих от конкретной модели. В данном случае — расход топлива на 100 км. Имеет связь с таблицей cars по FK models_id_car_fkey. У таблицы имеются следующие столбцы:
	id_model smallserial PRIMARY KEY (данный тип данных обусловлен тем, что количество моделей автомобилей скорее всего не превысит максимальное значение данного типа данных).
	model varchar NOT NULL (тип данных для хранения текстовых значений моделей; данный атрибут должен быть всегда заполнен).
	gasoline_consumption NUMERIC(3, 1) NULL (так как салон продает только легковые автомобили, установлено ограничение на числовое значение. Оно не может содержать больше 2 знаков перед запятой. Значение может быть пустым для электрокаров).
	id_car SMALLINT REFERENCES car_shop.cars(id_car) (выбор данного типа данных обусловлен типом PK для таблицы cars).
Таблица customers создана для хранения данных о покупателях:
	id_customer serial PRIMARY KEY (предполагается, что количество клиентов превысит максимальное значение для типа данных smallserial в будущем).
	first_name varchar (тип данных для хранения текстовых значений имен покупателей).
	last_name varchar (тип данных для хранения текстовых значений фамилий покупателей).
	phone varchar (тип данных для хранения текстовых значений номеров телефонов покупателей).
Таблица countries_of_origin содержит страны-производители автомобилей. В ней хранятся следующие атрибуты:
	id_brand serial PRIMARY KEY (количество стран в мире не превышает 32767).
	brand_origin varchar (тип данных для хранения наименования страны производителя).
Таблица sales содержит информацию о продажах автомобилей:
	id serial PRIMARY KEY (тип данных обусловлен тем, что количество продаж будет увеличиваться и превысит максимальное значение для smallserial).
	id_model SMALLINT REFERENCES car_shop.models(id_model) (тип данных обусловлен типом данных PK из таблицы models).
	id_color SMALLINT REFERENCES car_shop.colors(id_color) (тип данных обусловлен типом данных PK из таблицы colors).
	price numeric(9, 2) (тип данных обусловлен ограничением стоимости автомобиля, а именно числовое значение не может иметь более 7 знаков перед запятой).
	"date" date (для хранения информации о дате покупки).
	id_customer integer REFERENCES car_shop.customers(id_customer) (тип данных обусловлен типом данных PK из таблицы customers).
	discount numeric(4, 2) (размер скидки не может содержать больше 2 знаков после запятой).
	id_brand SMALLINT REFERENCES car_shop.countries_of_origin(id_brand) NULL (тип данных обусловлен типом данных PK из таблицы countries_of_origin, может принимать значение NULL).
